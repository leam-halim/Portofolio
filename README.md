# Portofolio V5

Halo semua! 👋

Perkenalkan, saya **Gulamin Halim**. Pada kesempatan ini, saya ingin membagikan proyek situs web portofolio yang telah saya kembangkan. Proyek ini dibangun menggunakan React dan Supabase, yang dilengkapi dengan situs publik serta dasbor admin.

> [!NOTE]  
> Proyek/template portofolio ini didasarkan dan dikembangkan menggunakan template milik **EkiZR** dari repositori [Portofolio_V5](https://github.com/EkiZR/Portofolio_V5.git).

**Live Demo:** [https://gulamin.com](https://gulamin.com)

---

## 🛠️ Teknologi yang Digunakan

Proyek ini dibangun menggunakan teknologi web modern berikut:

  - **ReactJS** - Framework frontend
  - **Tailwind CSS** - Framework CSS berbasis utility-first
  - **Supabase** - Backend untuk data portofolio, sertifikat, dan sistem komentar
  - **AOS** - Library animasi saat scroll (Animate On Scroll)
  - **Framer Motion** - Library animasi
  - **Lucide** - Library ikon
  - **Material UI** - Library komponen React
  - **SweetAlert2** - Dialog alert yang interaktif dan menarik

## Memulai Proyek

### Prasyarat

- Node.js `>= 14.x`
- npm atau yarn

### 1. Kloning & Instalasi

```bash
git clone https://github.com/gulaminhalim/Portofolio_V5.git
cd Portofolio_V5
npm install
```

> Jika Anda mengalami masalah kecocokan dependensi (peer dependency): `npm install --legacy-peer-deps`

### 2. Environment Variables

Buat sebuah file `.env` di direktori utama (root):

```env
VITE_SUPABASE_URL=url-proyek-supabase-anda
VITE_SUPABASE_ANON_KEY=anon-key-supabase-anda
```

> Anda dapat menemukan nilai-nilai ini di dasbor Supabase Anda pada bagian **Settings → API**.  
> ⚠️ Jangan pernah meng-commit file `.env` ke repositori Git Anda — pastikan file ini terdaftar di `.gitignore`.

### 3. Konfigurasi Client Supabase (`src/supabase.js`)

```javascript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseKey = import.meta.env.VITE_SUPABASE_ANON_KEY

if (!supabaseUrl || !supabaseKey) {
  throw new Error('Supabase credentials missing. Check your .env file.')
}

export const supabase = createClient(supabaseUrl, supabaseKey)
```

### 4. Pengaturan Database

Buka proyek Supabase Anda → **SQL Editor** → jalankan skrip SQL berikut (cukup dijalankan sekali saja):

```sql
-- ============================
-- TABEL-TABEL
-- ============================
CREATE TABLE public.projects (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title text,
  description text,
  img text,
  link text,
  github text,
  features jsonb,
  tech_stack jsonb,
  is_published boolean DEFAULT true,
  order_index int DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE public.certificates (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  img text,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE public.portfolio_comments (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  content text NOT NULL,
  user_name text NOT NULL,
  profile_image text,
  is_pinned boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE public.profiles (
  id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  username text UNIQUE NOT NULL,
  role text NOT NULL CHECK (role IN ('admin', 'user')),
  created_at timestamptz DEFAULT now()
);

-- ============================
-- RLS (Row Level Security)
-- ============================
ALTER TABLE public.projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.certificates ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.portfolio_comments ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "public read projects"
ON public.projects FOR SELECT USING (true);

CREATE POLICY "public read certificates"
ON public.certificates FOR SELECT USING (true);

CREATE POLICY "public read comments"
ON public.portfolio_comments FOR SELECT USING (true);

CREATE POLICY "public insert comment"
ON public.portfolio_comments FOR INSERT
WITH CHECK (is_pinned = false);

CREATE POLICY "admin manage projects"
ON public.projects FOR ALL
USING (
  EXISTS (
    SELECT 1 FROM public.profiles
    WHERE id = auth.uid() AND role = 'admin'
  )
);

CREATE POLICY "admin manage certificates"
ON public.certificates FOR ALL
USING (
  EXISTS (
    SELECT 1 FROM public.profiles
    WHERE id = auth.uid() AND role = 'admin'
  )
);

CREATE POLICY "admin manage comments"
ON public.portfolio_comments FOR UPDATE, DELETE
USING (
  EXISTS (
    SELECT 1 FROM public.profiles
    WHERE id = auth.uid() AND role = 'admin'
  )
);

-- ============================
-- STORAGE (Penyimpanan Gambar)
-- ============================
INSERT INTO storage.buckets (id, name, public)
VALUES ('project-images', 'project-images', true)
ON CONFLICT DO NOTHING;

CREATE POLICY "admin upload project images"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'project-images'
  AND EXISTS (
    SELECT 1 FROM public.profiles
    WHERE id = auth.uid() AND role = 'admin'
  )
);

CREATE POLICY "public read project images"
ON storage.objects FOR SELECT
USING (bucket_id = 'project-images');

INSERT INTO storage.buckets (id, name, public)
VALUES ('certificate-images', 'certificate-images', true)
ON CONFLICT DO NOTHING;

CREATE POLICY "admin upload certificate images"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'certificate-images'
  AND EXISTS (
    SELECT 1 FROM public.profiles
    WHERE id = auth.uid() AND role = 'admin'
  )
);

CREATE POLICY "public read certificate images"
ON storage.objects FOR SELECT
USING (bucket_id = 'certificate-images');
```

### 5. Mengaktifkan Fitur Realtime (Untuk Komentar)

Buka dasbor Supabase → **Table Editor → portfolio_comments → Enable Realtime**.

### 6. Membuat Akun Admin

**Langkah 1** — Buka bagian **Authentication → Users → Add User** di Dasbor Supabase, lalu salin User ID (UUID) yang dibuat.

**Langkah 2** — Jalankan perintah SQL berikut di SQL Editor (ganti `'USER_UUID'` dengan ID yang telah Anda salin):

```sql
INSERT INTO public.profiles (id, username, role)
VALUES ('USER_UUID', 'gulamin', 'admin');
```

### 7. Menjalankan di Lokal (Development)

```bash
npm run dev
```

Buka halaman `http://localhost:5173` di peramban (browser) Anda.

## Halaman & Fitur Utama

### Tampilan Publik (Pengunjung)
- **Home** — Bagian Hero, perkenalan diri, dan keahlian (skills).
- **Projects** — Menampilkan daftar proyek yang dipublikasikan beserta modal detail proyek.
- **Certificates** — Galeri sertifikat yang dimiliki.
- **Comments** — Melihat komentar yang ada dan mengirim komentar baru (menggunakan nama serta opsi foto profil).

### Dasbor Admin
- **Halaman Login** — Otentikasi menggunakan email dan kata sandi melalui Supabase Auth.
- **Dashboard** — Panel ringkasan utama setelah berhasil masuk.
- **Projects** — Membuat, menyunting, dan menghapus proyek; serta mengelola gambar, tautan, fitur, teknologi yang dipakai, status rilis, dan urutan tampil.
- **Certificates** — Mengunggah dan menghapus gambar sertifikat.
- **Comments** — Memantau seluruh komentar, menyematkan komentar penting, dan menghapus komentar yang tidak pantas.

## Solusi Masalah (Troubleshooting)

- Pastikan Node.js sudah terinstall dengan versi yang sesuai dan Anda berada di direktori proyek yang benar.
- Periksa kembali nilai-nilai pada `.env` dan jalankan ulang server development apabila terjadi perubahan konfigurasi.
- Jika RLS memblokir aksi tertentu, pastikan baris data admin Anda sudah terdaftar di tabel `profiles`.
- Bersihkan cache browser jika halaman menampilkan data lama yang tidak diperbarui.
